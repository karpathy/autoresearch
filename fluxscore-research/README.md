# FluxScore Autoresearch

Autonomous overnight ML experimentation for FluxScore — Innflux's credit scoring model.

Adapted from [karpathy/autoresearch](https://github.com/karpathy/autoresearch). Same loop
architecture (program.md + git-native + autonomous agent), different internals.

## Key differences from the original

| | karpathy/autoresearch | fluxscore-research |
|---|---|---|
| Model | GPT / PyTorch LLM | scikit-learn credit model |
| Hardware | H100 GPU required | CPU-only laptop |
| Metric | `val_bpb` (lower = better) | `auc` (higher = better) |
| Time budget | 5 minutes | 2 minutes |
| Data | Open text corpus | Synthetic borrower profiles |
| Real data | Public | NEVER — real data stays in TEE |
| Evaluation | `prepare.py` fixed harness | `_evaluate()` in `experiment.py` fixed harness |

## Quick start

```bash
# 1. Install deps
pip install -e fluxscore-research/

# 2. Generate synthetic data (one-time)
python fluxscore-research/prepare.py

# 3. Verify baseline runs
python fluxscore-research/experiment.py

# 4. Start the agent loop (point your agent at program.md)
# The agent modifies experiment.py below the AGENT ZONE marker,
# commits, runs, checks grep "^auc:" run.log, keeps or reverts.
```

## What the agent modifies

Only `experiment.py` — and only the code below `# AGENT ZONE`. The fixed harness above it
enforces the 2-minute budget (`signal.alarm(120)`), loads the data, and defines the evaluation
function. The agent cannot touch these.

## Promotion path

When the agent finds a model that beats baseline by > 0.02 AUC on synthetic holdout:

1. Agent flags it in `results.tsv` with `[PROMOTE: validate in TEE]`.
2. Human engineer manually validates the architecture on real borrower data inside the TEE.
3. If it holds up, deploy as an updated `FluxScoreOracle` contract.

The agent never touches real data. Decoupling research from production data is the key design decision.

## File layout

```
fluxscore-research/
├── README.md         — this file
├── program.md        — research brief (the agent reads this)
├── prepare.py        — synthetic data generator (fixed, do not modify)
├── experiment.py     — agent's playground (fixed harness + agent zone)
├── results.tsv       — experiment log (untracked by git)
├── pyproject.toml    — deps: scikit-learn, pandas, lgbm, xgboost, imbalanced-learn
└── train.parquet     — generated by prepare.py (gitignored)
    holdout.parquet   — generated by prepare.py (gitignored)
```

## Running overnight

Same as original autoresearch — point your Claude agent at `program.md` and tell it to
start the experiment loop. 2-minute experiments = ~30/hour = ~240 over a typical sleep.

The agent commits each change so the entire experiment history is in git.
