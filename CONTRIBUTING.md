# Contributing to autoresearch

Thanks for your interest in contributing!

## Ways to Contribute

### 🐛 Bug Reports
- Check existing issues first
- Include your GPU, Python version, and error message
- Provide steps to reproduce

### 🔧 Code Changes
- Keep changes minimal and focused
- Follow existing code style (4-space indent, no trailing whitespace)
- Test on at least one GPU before submitting

### 📝 Documentation
- Fix typos, clarify instructions
- Add troubleshooting tips from your experience
- Update Notable forks with working alternatives

### 💡 Experiment Results
- Share successful experiments in Discussions
- Document what worked and what didn't

## Pull Request Process

1. Fork the repo
2. Create a feature branch (`git checkout -b feat/my-feature`)
3. Make your changes
4. Test: `uv run train.py` should complete without errors
5. Commit with a clear message
6. Open a PR with description of what and why

## Code Style

- Python: 4-space indentation, following existing patterns
- Markdown: Use fenced code blocks with language hints
- Keep it simple — this repo values simplicity

## Questions?

Open a Discussion or check existing issues.
