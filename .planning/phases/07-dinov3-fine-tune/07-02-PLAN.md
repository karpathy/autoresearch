1. **Overview** -- "You are fine-tuning DINOv3 ViT-H+ (840M) on the product dataset using LoRA and supervised InfoNCE contrastive loss. Your goal: maximize recall@1 on the validation set."

2. **Hard Constraints (NEVER violate):**
   - NEVER edit prepare_dino.py
   - NEVER add new dependencies (peft, transformers, torch are available)
   - NEVER exceed EPOCHS=10 per experiment
   - NEVER stop the loop -- always continue to the next experiment
   - NEVER disable gradient checkpointing (OOM risk on 24GB GPU)
   - NEVER set BATCH_SIZE > 16 (OOM risk -- use GRADIENT_ACCUMULATION_STEPS for larger effective batch)

3. **Search Space** -- Document every tunable constant from train_dino.py with:
   - Current default value
   - Safe range to explore
   - What it controls and expected effect

   Constants to document:
   - LORA_R: default 16, range [4, 64], higher = more capacity but more VRAM
   - LORA_ALPHA: default 32, typically 2x LORA_R
   - LORA_DROPOUT: default 0.05, range [0.0, 0.2]
   - LORA_TARGET_MODULES: default ["q_proj", "v_proj"], can add "k_proj", "out_proj", MLP modules
   - BATCH_SIZE: default 8, range [4, 16], VRAM limited
   - GRADIENT_ACCUMULATION_STEPS: default 16, range [4, 64], higher = larger effective batch
   - LR: default 2e-4, range [1e-5, 1e-3]
   - WEIGHT_DECAY: default 0.01, range [0.0, 0.1]
   - WARMUP_RATIO: default 0.1, range [0.0, 0.3]
   - TEMPERATURE: default 0.07, range [0.03, 0.2], CRITICAL for contrastive learning

4. **Experiment Strategy (prioritized):**
   - Priority 1: Temperature tuning (most impactful for contrastive learning)
   - Priority 2: Learning rate sweep (2e-5 to 5e-4)
   - Priority 3: LoRA rank exploration (4, 8, 16, 32)
   - Priority 4: Effective batch size (64, 128, 256 via gradient accumulation)
   - Priority 5: Target module expansion (add k_proj, out_proj, MLP)
   - Priority 6: Augmentation changes (modify crop scale, add color jitter)

5. **Workflow** -- Same as main autoresearch:
   - Read results.tsv to see what has been tried
   - Modify train_dino.py constants
   - Run: `cd dino_finetune && python train_dino.py`
   - Read output for METRIC line
   - If improved: git commit and keep
   - If regressed: git reset --hard and try different direction
   - Repeat forever

6. **Common Pitfalls:**
   - Temperature too low (< 0.03) causes NaN loss
   - Temperature too high (> 0.5) makes loss insensitive
   - Batch size too large causes OOM (use gradient accumulation instead)
   - LoRA rank too high with target module expansion causes OOM
   - Forgetting to check peak_vram_mb before committing

7. **Output Format** -- Explain the METRIC and RESULT lines that prepare_dino.py/train_dino.py produce, and how results.tsv is structured.
