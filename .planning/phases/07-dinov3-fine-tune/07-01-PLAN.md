4. `extract_cls_embedding(model_output) -> Tensor[B, 1280]` -- Extract CLS token from model output. DINOv3 uses index 0 of last_hidden_state (then 4 register tokens, then patches). L2-normalize the result. This is a critical function -- document the register token offset.

5. `evaluate_dino(model, val_loader, device) -> dict` -- Compute recall@1 and mean_cosine on validation set (per D-08). Iterate val_loader, extract CLS embeddings via extract_cls_embedding, compute pairwise cosine similarity, find nearest neighbor recall@1, compute mean intra-class cosine. Return {"recall@1": float, "mean_cosine": float, "combined": float} where combined = 0.5 * recall@1 + 0.5 * mean_cosine.

6. `save_adapter(model, output_dir=ADAPTER_OUTPUT_DIR)` -- Save LoRA adapter weights via model.save_pretrained(). Create output_dir if needed. Log saved path.

7. `load_finetuned_model(adapter_path=ADAPTER_OUTPUT_DIR, device="cuda")` -- Load base model + LoRA adapter via PeftModel.from_pretrained(). Set to eval mode. Return merged model ready for inference.

8. `collate_fn(batch)` -- Collate function for DataLoader that stacks images and labels.

9. `set_seed(seed=42)` -- Set random seeds for reproducibility (torch, numpy, random).

**Training functions to implement:**

1. `info_nce_loss(embeddings, labels, temperature=TEMPERATURE)` -- Supervised InfoNCE contrastive loss per D-02. Input: [B, D] L2-normalized embeddings, [B] integer product labels. Same-product images are positives, different products are negatives. Handle edge case where some anchors have no positives in the batch. Return scalar loss.

2. `build_optimizer(model)` -- Create AdamW optimizer for only the trainable (LoRA) parameters. Use LR and WEIGHT_DECAY constants.

3. `build_scheduler(optimizer, num_training_steps)` -- Cosine LR schedule with linear warmup for WARMUP_RATIO fraction of total steps. Use torch.optim.lr_scheduler (not transformers -- keep dependencies minimal).

4. `train_one_epoch(model, train_loader, optimizer, scheduler, scaler, device, epoch)` -- One epoch of training. For each batch: forward pass through model, extract CLS embeddings via extract_cls_embedding, compute info_nce_loss, backward with gradient scaling (bf16 autocast), accumulate gradients for GRADIENT_ACCUMULATION_STEPS, then optimizer.step(). Log loss every 50 steps. Return average epoch loss.

5. `main()` -- Full training loop:
   - set_seed(SEED)
   - Load base model via load_base_model()
   - Inject LoRA via peft: LoraConfig(r=LORA_R, lora_alpha=LORA_ALPHA, lora_dropout=LORA_DROPOUT, target_modules=LORA_TARGET_MODULES, bias="none") then get_peft_model()
   - Print trainable parameters count (should be ~0.2% of total)
   - Enable gradient checkpointing if USE_GRADIENT_CHECKPOINTING
   - Build train/val datasets and DataLoaders (use get_image_processor, build_dataset, collate_fn)
   - Build optimizer and scheduler
   - Training loop for EPOCHS epochs:
     * train_one_epoch()
     * If epoch % EVAL_EVERY_N_EPOCHS == 0: evaluate_dino()
     * Track best combined metric, save_adapter when improved
   - Print final results in greppable format: "RESULT: recall@1={:.4f} mean_cosine={:.4f} combined={:.4f} peak_vram_mb={}"
   - Print "METRIC: {combined:.6f}" on its own line (for autoresearch harness)

**Important details:**
- Use torch.amp.autocast("cuda", dtype=torch.bfloat16) for forward passes
- Use torch.amp.GradScaler for mixed precision training (even with bf16, scaler handles edge cases)
- Log peak VRAM at end of first epoch for profiling

- Pin num_workers=4 for DataLoader
