  **9. Domain Context: ReID Knowledge Distillation**
  Brief primer so the agent understands the domain:
  - ReID (re-identification): Given a query image, find matching products/people in a gallery using embedding similarity.
  - Knowledge distillation: A large teacher model (ONNX + DINOv2) produces high-quality embeddings. A small student model (LCNet050) learns to match them while remaining edge-deployable.
  - The combined metric balances two objectives: recall@1 (retrieval accuracy -- can the model find the right product?) and mean_cosine (teacher alignment -- does the student agree with the teacher?).
  - The student model processes images -> backbone (LCNet050) -> projection head -> L2-normalized 256-dim embedding.
  - Training uses multiple losses: distillation (cosine with teacher), ArcFace (classification margin), VAT (adversarial robustness), separation (push blacklist away from whitelist).

  **10. Search Space Reference** (per D-03 through D-06)

  **11. ReID Experiment Playbook** (per D-02: directional guidance, not rigid script)
  Three priority tiers with directional hints (not exact values):

  **Tier 1 -- High Impact, Low Risk (try first):**
  - Loss weight ratios: Adjust balance between distillation, ArcFace, and separation losses. The default ArcFace weight (0.05) is conservative -- experiment in range 0.01-0.2.
  - Learning rate and schedule: Try cosine annealing with warmup. Experiment with warmup fraction (5-20% of epochs). Try lower LR (1e-2 to 1e-3) with longer warmup.
  - Unfreeze epoch and stages: Currently unfreezes at epoch 5. Try earlier (epoch 2-3) or later (epoch 7-8). Try unfreezing fewer or more stages. Use lower LR for backbone than for head.

  **Tier 2 -- Medium Impact, Medium Risk:**
  - Projection head design: Add hidden layers between backbone and embedding. Try BN->ReLU->Linear patterns. Try dropout. Keep final dim = 256.
  - Augmentation parameters: Tune quality degradation intensity (prob, downsample ratio, quality range). Add/adjust color jitter, random erasing. NEVER remove quality degradation entirely.
  - ArcFace margin and scale: Tune ARCFACE_M (try 0.3-0.7) and ARCFACE_S (try 16-64). These control class separation sharpness.
  - VAT regularization: Currently disabled (weight=0). Try enabling with small weight (0.01-0.1). May improve embedding robustness.
  - Optimizer choice: Default is SGD with momentum. Try AdamW with lower LR (1e-3 to 1e-4).

  **Tier 3 -- High Impact, Higher Risk (try when Tier 1/2 plateau):**
  - Different backbone: Try other lightweight models from timm registry. MUST remain edge-deployable -- check params and GFLOPs. Examples: mobilenetv3_small, efficientnet_lite0. Always verify parameter count is comparable to LCNet050.
  - Batch size changes: Increase for better gradient estimates but watch VRAM. Decrease if hitting OOM.
  - ArcFace phaseout: Try phasing out ArcFace loss after epoch N to let distillation dominate in final epochs.
  - Novel loss ideas: Circle loss, subcenter ArcFace, proxy-anchor loss. Implement from scratch using only torch (no new deps). These are advanced -- only try if Tier 1/2 ideas are exhausted.

  **12. When You Are Stuck**
  Three escalation levels (per 03-RESEARCH.md Decision 3):
  1. Minor plateau (3-5 no-improvement experiments): Switch to a different parameter dimension. If you were tuning LR, switch to loss weights. If tuning losses, try augmentation.
  2. Medium plateau (5-10 no-improvement experiments): Jump to a different tier. Re-read results.tsv and identify the single best configuration. Try combining it with ideas from unexplored dimensions.
  3. Major plateau (10+ no-improvement experiments): Try radical changes -- different backbone, novel loss function, extreme augmentation changes. Re-read train.py carefully for overlooked opportunities. Re-read this playbook.

  **13. Reading results.tsv for History Reasoning** (AGNT-05)
  Explicit instructions:
  - Before EVERY experiment, read results.tsv: `cat results.tsv`
  - Analyze: How many experiments run? What is the current best combined_metric? Which changes improved it? Which hurt? Which crashed?
  - Look for patterns: "LR changes helped", "ArcFace weight increases hurt", "VAT consistently crashes"
  - Use decomposed metrics: If recall@1 improved but mean_cosine dropped, the change helped retrieval but hurt teacher alignment. Choose next experiment to address the weaker metric.
  - If multiple crashes on similar ideas, skip that direction. Move to a completely different parameter dimension.
  - Keep descriptions informative in results.tsv so you can reason about them later. Bad: "try stuff". Good: "reduce ArcFace weight from 0.05 to 0.02, increase LR to 0.15"

  **14. Crash Handling** (extends experiment loop)
  - If run crashes, first check: `tail -n 50 run.log`
  - If OOM: your next experiment MUST reduce compute, not try the same thing with a minor tweak. Reduce batch size, reduce model size, or simplify.
  - If bug (typo, import error): fix and re-run. This does not count as a failed experiment.
  - If 3+ consecutive crashes on the same idea: skip it entirely and move to a different direction.
  - VRAM budget rule: If peak_vram_mb > 22000 on a successful run, do NOT increase batch size or model size further. You are near the 24GB limit.
