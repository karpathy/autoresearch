<task type="auto">
  <name>Task 2: Execute one full autonomous agent cycle (VALD-02)</name>
  <files>train.py, results.tsv, run.log</files>
  <read_first>
    - program.md (the agent instructions that define the autonomous loop protocol)
    - results.tsv (current state after baseline -- the agent reads this to decide what to try next)
    - train.py (current baseline state -- the agent modifies this)
  </read_first>
  <action>
    Demonstrate one complete autonomous cycle: read history, modify train.py, run experiment, evaluate, keep or discard.

    Steps:
    1. Read results.tsv to see the baseline metric (simulating agent behavior per AGNT-05).
    2. Make a small, safe modification to train.py -- change one module-level constant. Good candidates:
       - Adjust a loss weight slightly (e.g., if there is a distill_loss_weight, change it by 10-20%)
       - Adjust learning rate slightly
       - The change should be small enough to not crash but large enough to produce a different metric
    3. Commit the change: `git add train.py && git commit -m "experiment: [description of change]"`
    4. Run the experiment: `python train.py > run.log 2>&1`
    5. After completion, verify the experiment was evaluated and logged:
       - results.tsv has a new row (row count increased by 1)
       - The new row has the correct commit hash
       - The new row has a valid combined metric
    6. Verify the keep/discard logic worked correctly:
       - If metric improved: status should be "kept", git state should have the commit
       - If metric regressed: status should be "discarded", train.py should have been reverted via git reset
    7. Verify git state matches the results.tsv status:
       - `git log --oneline -3` should show the correct history
       - `git diff` should be clean (no uncommitted changes)

    This task proves the full agent loop works. The specific metric result does not matter -- what matters is that the cycle completed and the keep/discard logic functioned correctly.
  </action>
  <verify>
    <automated>test $(awk 'NR>1' results.tsv | wc -l) -ge 2 && echo "PASS: at least 2 experiment rows" || echo "FAIL: fewer than 2 rows"</automated>
  </verify>
  <acceptance_criteria>
    - results.tsv has at least 2 data rows (baseline + experiment): `awk 'NR>1' results.tsv | wc -l` returns >= 2
    - The second row has a valid combined metric (non-zero, non-NaN)
    - The second row has status "kept" or "discarded" (not "crash")
    - Git state is consistent: `git diff --stat` is empty (clean working tree)
    - If status is "kept": `git log --oneline -1` shows the experiment commit
    - If status is "discarded": `git log --oneline -1` shows the baseline commit (experiment was reset)
  </acceptance_criteria>
  <done>One full autonomous cycle completed: train.py was modified, experiment ran, metrics were evaluated, and the result was correctly kept or discarded with proper git state management.</done>
</task>
